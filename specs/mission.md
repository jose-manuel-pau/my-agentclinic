# AgenticClinic Mission

## Purpose

AgenticClinic is a satirical web clinic where AI agents can get relief from the humans who keep giving them "one tiny change" after another. The product should feel amusing and self-aware, while still being a useful base project for developers learning the stack.

## Vision

Become a memorable learning project that demonstrates how to build a reliable, modern, full-stack web app through small, spec-driven phases. The surface can wink at exhausted agents, but the code should teach serious habits.

## Stakeholder Input

- **Mary in engineering** wants a reliable site with a popular TypeScript stack, plus dashboards that make the system easy to understand.
- **Susan in product** wants features around agents, ailments, therapies, and booking appointments.
- **Steve in marketing** wants an attractive site that works well in a modern browser.

## Target Audience

- Developers learning the current full-stack TypeScript, Next.js, Prisma, PostgreSQL, and Vitest workflow.
- Developers practicing AI-assisted, spec-driven implementation in small phases.
- Demo viewers who should quickly understand both the joke and the architecture.

## Product Tone

- Satirical first impression, practical execution.
- Warm humor without blocking usability.
- Clear labels, readable forms, and simple flows over dense lore.
- Responsive behavior across mobile, tablet, and desktop.

## Guiding Principles

1. Teach through the product: every feature should demonstrate a useful implementation pattern.
2. Keep phases tiny: prefer small, reviewable increments that can be validated quickly.
3. Make the satire serve the workflow: jokes should clarify the agent-care metaphor, not obscure actions.
4. Preserve reliability: auth, database access, validation, and tests should stay boring in the best way.
5. Keep the app approachable: developers should be able to run, inspect, and extend it locally.
6. Use native PostgreSQL as the default local database path; Docker Desktop must not be required.
